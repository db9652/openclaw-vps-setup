# Contributing

Thanks for considering a contribution! This project exists to make OpenClaw accessible to everyone.

## How to Contribute

1. **Fork** the repo
2. **Create a branch** for your feature: `git checkout -b feature/my-improvement`
3. **Make your changes**
4. **Test** — build the Docker image locally: `docker compose build`
5. **Commit** with a clear message
6. **Open a PR** explaining what you changed and why

## Ideas for Contributions

- 🐛 Bug fixes (Dockerfile, scripts, docs)
- 📝 Better documentation or translations
- 🔒 Security improvements
- 📱 Additional channel setup guides
- 🧠 Improved workspace templates
- 🐳 Docker optimizations (smaller image, faster builds)
- 🧪 CI/CD pipeline (GitHub Actions for build testing)
- 🌍 Localization (non-English setup guides)

## Guidelines

- Keep it simple — this project is for beginners
- Don't add secrets or API keys (even fake ones that look real)
- Test your Docker build before submitting
- Security defaults should stay strict (don't weaken them for convenience)

## Questions?

Open an issue — happy to discuss before you start coding.
